1. Giới thiệu RARZIP Tool

RARZIP Tool khôi phục mật khẩu file nén RAR (RAR3, RAR5) và ZIP bằng phương pháp vét cạn hoặc pattern. Chạy trực tiếp trên Windows, không cần cài đặt.

Hỗ trợ: .rar (RAR3, RAR5), .zip (bao gồm AES-256)

Backend: WinRAR (UnRAR.exe) hoặc 7-Zip (7z.exe)

💡 Tool tự động phát hiện WinRAR hoặc 7-Zip đã cài trên máy.

2. Tính năng cập nhật mới

✅ Hỗ trợ pass dài 10+ ký tự — không còn đơ

Phiên bản cũ bị đơ/crash khi đặt max length ≥ 10 vì cố load toàn bộ hàng tỷ candidates vào RAM. Bản mới dùng batch feeding: chia nhỏ 500K/đợt, workers xử lý xong đợt này mới lấy đợt tiếp.

✅ ETA hiển thị năm-tháng-ngày-giờ

Thay vì hiển thị số ngày khổng lồ khó đọc, giờ hiển thị dạng ~2y 3mo 15d 8h hoặc ~5d 14h tùy độ lớn.

✅ Cảnh báo khi candidates > 50 triệu

Popup cảnh báo kèm ước tính thời gian thực tế trước khi bắt đầu — giúp bạn quyết định có nên thu hẹp bộ ký tự không.

✅ GUI cập nhật ngay khi bắt đầu

Trước đây W1-W8 không hiển thị ngay — phải ấn tạm dừng rồi tiếp tục mới thấy. Đã sửa: GUI tự chờ và cập nhật ngay.

💡 Dù hỗ trợ pass 10+ ký tự, brute force toàn bộ charset 62 ký tự ở độ dài 10 cần ~52 năm. Dùng Pattern nếu biết một phần để giảm xuống vài giây.

3. Yêu cầu hệ thống

  • Windows 7/8/10/11 (64-bit khuyến nghị)
  • WinRAR hoặc 7-Zip đã cài đặt (để xử lý file RAR)
  • File ZIP thuần: không cần phần mềm thêm
⚠️ Nếu không có WinRAR hoặc 7-Zip, tool chỉ hỗ trợ file ZIP (không hỗ trợ RAR).

5. Chọn file và cấu hình

  1. Nhấn 📂 Browse hoặc kéo thả file RAR/ZIP vào ô chọn file
  2. Kiểm tra đường dẫn UnRAR.exe / 7z.exe (tự động điền nếu đã cài)
  3. Chọn số luồng X1-X8 theo sức máy
  4. Cấu hình bộ ký tự và độ dài

5. Chế độ Vét cạn (Brute Force)

Thử tất cả tổ hợp ký tự trong bộ ký tự đã chọn.

  1. Chọn bộ ký tự: 0-9, a-z, A-Z, ký tự đặc biệt (@) hoặc Tất cả
  2. Đặt độ dài tối thiểu và tối đa
  3. Nhấn ▶️ Bắt đầu
💡 Ctrl+click để chọn nhiều bộ ký tự cùng lúc. Ví dụ: 0-9 + a-z.

6. Chế độ Pattern

Dùng khi biết một phần mật khẩu. Dấu * đại diện cho ký tự chưa biết.

Ví dụ:

  • abc* → thử abc0, abc1, ..., abcz (1 ký tự chưa biết)
  • 2024** → thử 202400, 202401, ..., 2024zz (2 ký tự chưa biết)
  • *pass* → thử 0pass0, 0pass1, ..., zpassz
💡 Pattern ưu tiên hơn cấu hình độ dài. Nếu nhập pattern thì độ dài bị bỏ qua.

7. Đa luồng X1-X8 & tạm dừng từng luồng

Chia danh sách mật khẩu thành các chunk, mỗi luồng xử lý 1 chunk song song — giống như mở nhiều app cùng lúc nhưng trong 1 app duy nhất.

  • X1 — 1 luồng, dùng 1 nhân CPU. FREE + PRO, phù hợp máy yếu hoặc CPU 2 nhân
  • X2 — 2 luồng, dùng 2 nhân CPU. Nhanh ~2x. Cần CPU ≥ 2 nhân
  • X3/X4/X5 — 3-5 luồng. Cần CPU ≥ 4 nhân, chỉ PRO
  • X6/X7/X8 — 6-8 luồng. Nhanh nhất. Cần CPU ≥ 6 nhân, chỉ PRO
💡 Kiểm tra số nhân CPU: Chuột phải Taskbar → Task Manager → Performance → CPU → xem "Cores". Chọn số luồng ≤ số nhân CPU thực tế.

Mỗi luồng hiển thị: W1-W8 với candidate đang thử, số đã thử / tổng, timer riêng, ETA và %. Có thể tạm dừng/tiếp tục từng luồng riêng lẻ bằng nút ⏸/▶.

⚠️ Dùng X8 trên CPU 4 nhân không nhanh hơn X4, thậm chí chậm hơn. Chọn số luồng ≤ số nhân CPU thực tế.

8. Mẹo và lưu ý

  • File RAR5 (WinRAR 5+) dùng AES-256 → chậm hơn RAR3
  • File ZIP với AES-256 cần 7-Zip để xử lý đúng
  • Biết mật khẩu là số → chỉ dùng bộ ký tự 0-9
  • Biết một phần → dùng Pattern để giảm không gian tìm kiếm
  • Tắt antivirus tạm thời nếu tool bị chặn (false positive)
  • Dùng X4 trên máy có CPU 4+ core để tối đa tốc độ
